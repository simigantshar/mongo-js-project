Toys API

HTTP Method: Route Description Parameters / Request Body
get /toys returns list of toys matching search query's
  • page page number to display (default: 1).
  • search (optional) the search query to match the name and info fields (default: null)
  • category (optional) cat query displays items that match the category
  • link: https://mongo-js-simi-gan.cyclic.app/toys/?page=1&s=car
  • link: https://mongo-js-simi-gan.cyclic.app/toys/?page=1&cat=RC
get /toys/prices returns list of toys filtered by price range
  • page page number to display page (default: 1)
  • min min query to display items equall or greater than min (optional)
  • max max query to display items equall or lower than max (optional)
  • link: https://mongo-js-simi-gan.cyclic.app/toys/prices/?min=30&max=200
get /toys/single/:id returns toy matching parameter
  • id parameter that returns matching id
  • link: https://mongo-js-simi-gan.cyclic.app/toys/single/:id
post /toys adds a new toy.
token is required!
  • Request Body: adds data from body.
  • Toy Schema:
    {
    name: String,
    info: String,
    category: String,
    img_url: String,
    price: Number
    }
put /toys/:edit updates toy matching parameter
Token is required!
  • id toy of matchinf id to update
delete /toys/:delete deletes toy matching parameter
  • id: id of item to delete
